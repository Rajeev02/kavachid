# Contributing to Kavach Shield SDK

First off, thank you for considering contributing to Kavach! It's people like you that make Kavach an incredibly secure and powerful zero-trust framework.

## How Can I Contribute?

### 1. Reporting Bugs
If you find a bug, please create an Issue on GitHub. Include:
* What OS/Platform you are running
* Steps to reproduce the bug
* Expected behavior vs Actual behavior

### 2. Suggesting Enhancements
We welcome feature requests! Please create an Issue outlining:
* The problem you are trying to solve
* Your proposed solution or API design

### 3. Submitting Pull Requests
Kavach strictly protects the `main` branch. **Direct pushes to `main` are disabled.**
All code changes must go through a Pull Request.

1. Fork the repository
2. Create a new branch (`git checkout -b feature/amazing-feature`)
3. Make your changes in the appropriate `sdks/` directory
4. Ensure the code compiles and tests pass
5. Commit your changes (`git commit -m 'feat: add amazing feature'`)
6. Push to the branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

## Core Philosophy
* **Security First:** Never compromise on security for convenience.
* **Native Integration:** Always utilize hardware-backed security (Secure Enclave, StrongBox) where possible.

By contributing to Kavach, you agree that your contributions will be licensed under its MIT License.
